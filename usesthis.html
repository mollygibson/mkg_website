My Setup.

I am a strong believer in using the right tools to get a job done. That's why when I stumbled upon The Setup, I couldn't pull myself away. I quickly found a number new tools that have since made my job easier. I am now contributing to the community and posting my setup.

  • Who are you, and what do you do?

    My name is Molly Gibson and I am a Computational & Systems Biology graduate student at Washington University in St. Louis in Gautam Dantas' Lab. I wear many different hats as a computational and bench scientist: I research, I code, I write, I tinker, I organize, I tweet, etc. I am a big multi-tasker and I am always looking for the most useful tools to be as organized and efficient as possible.

  • What hardware do you use?

    I use a family of Apple products that are all seamlessly synced in every way possible. At both work and home I have a 27-inch iMac. I like to travel light, yet always have my 11-inch MacBook Air and iPad Air 2 in tow. It seems redudant, but they are both small, lightweight and serve very different purposes. With my MacBook Air, I have access to all of my data and computing cluster and with my iPad2, I can keep organized and jot notes without the need for any additional notepads.

    3D-printing is becoming more common in scientific laboratories to print low-cost and customized lab ware. I've therefore recently started dabbling with 3D-printing and bought a Ultimaker for use in lab. It is a fairly reliable, open-hardware 3D-printing option. When shopping around, it had some of the highest ratings in it's price category, however, I've had some problems with plastic jamming. I enjoy tinkering, so it was a good addition to my setup, but might still be in the early stages for widespread use.

  • And what software?

    For coding and text editing, I used Emacs in iTerm2 almost exclusively. This was the setup I used when learning to code and it has worked well, so I haven't seen a need to upgrade/modify these tools (yet). I code mostly in Python now, but I have also coded C++ and Java extensively. I use Eclipse when coding large software projects with multiple classes and packages. I really want to learn how to use iPython Notebook, but haven't taken the time to get it setup yet. I use R for statistical work and generating pretty graphs and Adobe Illustrator for editing/generating figures.

    I keep an electronic lab notebook using a MediaWiki and MySQL backend. This is one of the most useful setups that I have as a researcher and I can't imagine going back to paper and pen lab notebook.

    I use Microsoft Office for most documents and presentations for collaboration purposes (track changes, end note, etc.). If it were up to me, I would write exclusively in LaTeX and generate all presentations using the Beamer class. I use TeXShop for previewing and typesetting LaTeX.

    For keeping track of the scientific literature, I use Papers2. It syncs nicely across devices, which will be a continual theme throughout my setup. I also like how it itegrates bibliographies into documents, although again, I use EndNote for collaboration puposes. I have tried to use Mendeley For Paper organization as well because of it's easy collaboration tools, however, it doesn't (or at least didn't) have an app for iPad. This is necessary for me, as I try to read most papers electronically.

    For syncing documents across devices, I use Dropbox. I have a large paid account, which I use as a remote hard disk. This also serves as a constant backup. I use a lot of other Apple apps that sync nicely through iCloud as well, including iCal and Contacts (although Contacts is not really a choice as I am a iPhone user).

    I'm still learning CAD design for 3D printing purposes, but so far I've been using a combination of AutoCAD Student Version, which is free and is installed locally, and AutoDesk123, which is free and can be used in your brower. Design is the biggest bottleneck in 3D printing for me so far, so I'm still looking for the best way to tackle this problem.

    I often use Things to keep track of my to-do list, however, I often find post-it note to-do lists creeping back into my routine. As much as I want to use technology and not paper, there is something rewarding about physically crossing items off a list.

    To surf the web and communicate with others, Google dominates my setup. I use Chrome to browse, Gmail for email, Google chat for messaging, and Google Hangout for voice/conference calls (which I prefer over Skype, but find myself using that many times as well).

    For notetaking, I use Notability on my iPad and Evernote on my computer. It would be really nice if Notability could interface and automatically send notes to Evernote. I've seen talk in forums about this, hopefully it is in the works.

    Other miscellaneous tools that I use include 1Password for password organization, Cyberduck as an SFTP client, VLC for media player, Spotify for music, and RunKeeper for running.

  • What would be your dream setup?

    I am pretty content with my hardware setup, however, in an ideal world, I would alter how I perform computational work slightly. Specifically, I would like to use tools that help me improve commenting and version control. Two tools that I want to start integrating into my setup soon in that respect include GitHub and IPython Notebook. I would also like a more reliable 3D printer, but I guess I'm getting picky now.
